# Task GH-003: Authentication System

## Objective
Implement a simple authentication system with single user login.

## Requirements

### 1. User Table and Model
- Use Laravel's default users table or create simplified version
- Single user credentials stored in database or config
- Fields: username, password (hashed)

### 2. Login Page
- Route: `/login`
- Clean, centered login form with Bootstrap 5.3
- Fields:
  - Username input field
  - Password input field (type="password")
  - Login button
- Show validation errors if credentials are incorrect
- Redirect to home page after successful login

### 3. Authentication Logic
- Use Laravel's built-in authentication
- Implement login controller method
- Store session after successful authentication
- Implement logout functionality
- NO FORGOT PASSWORD
- NO USER REGISTRATION OPTION
- NO USER VERFICATION
- ONLY LOGIN PAGE
- Create artisan commands user {add, remove, list}

### 4. Middleware Protection
- Protect all routes except login with authentication middleware
- Redirect unauthenticated users to login page
- Redirect authenticated users away from login page to home

### 5. User Seeder (Optional)
- Create seeder for default admin user
- Document default credentials in setup documentation

## Acceptance Criteria
- [ ] Login page displays correctly with Bootstrap 5.3 styling
- [ ] User can log in with correct credentials
- [ ] Invalid credentials show appropriate error message
- [ ] Authenticated users can access protected routes
- [ ] Unauthenticated users are redirected to login
- [ ] Session management works correctly
- [ ] Logout functionality works

## Technical Notes
- Use Laravel's Auth facade and middleware
- Hash passwords with bcrypt
- Use CSRF protection on login form
- Consider using `auth` middleware group
