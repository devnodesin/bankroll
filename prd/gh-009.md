# Task GH-009: Category Management Feature

## Objective
Implement functionality to view, add, and manage transaction categories.

## Requirements

### 1. Category Management UI
- Add "Manage Categories" button/link in navigation or settings
- Opens modal or separate page showing category list
- Display two sections:
  - **Predefined Categories**: Non-editable system categories
  - **Custom Categories**: User-created categories (can be deleted)

### 2. Category List Display
- Table or list showing all categories
- Columns:
  - Category Name
  - Type (System/Custom)
  - Action buttons (Delete for custom categories only)
- Use Bootstrap styling (badges, list-group, or table)
- Visual distinction between system and custom categories

### 3. Add Category Feature
- Input field for new category name
- "Add Category" button
- Validation:
  - Required field
  - Unique name (case-insensitive)
  - Reasonable length (max 50 characters)
  - No special characters (optional)
- Show success message after adding
- New category appears in list immediately

### 4. Delete Category Feature
- Delete button only for custom categories
- Confirmation dialog before deletion
- Check if category is in use by transactions
- If in use:
  - Option 1: Prevent deletion with warning
  - Option 2: Allow deletion and unassign from transactions
- Show success message after deletion

### 5. Controller Logic
- Create CategoryController with CRUD methods
- Index: List all categories
- Store: Add new custom category
- Destroy: Delete custom category
- Validation on backend
- Protect system categories from modification

### 6. Integration with Transaction Table
- Updated category list immediately available in transaction Notes dropdown
- No page refresh needed (use AJAX)

## Acceptance Criteria
- [ ] Category management page/modal displays correctly
- [ ] All predefined categories are listed
- [ ] System categories cannot be deleted
- [ ] Users can add custom categories
- [ ] Category name validation works
- [ ] Duplicate categories are prevented
- [ ] Custom categories can be deleted
- [ ] Deletion confirmation works
- [ ] Category deletion handles transactions appropriately
- [ ] New categories appear in transaction dropdown
- [ ] All operations provide user feedback

## Technical Notes
- Mark predefined categories with `is_custom = false` in database
- Use soft deletes if you want to preserve category history
- Consider adding category usage count in list
- Use Bootstrap modals for confirmations
- Implement AJAX for add/delete operations
- Validate uniqueness case-insensitively
